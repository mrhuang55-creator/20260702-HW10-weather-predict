## 🛠️ 第四階段實作計畫書 (Implementation Plan)

### 1. 變更範圍與修改檔案

#### [MODIFY] [weather.py](file:///f:/HW10-20260702-天氣預報/weather.py)
- **資料庫檢查與自動升級**：在 `init_db()` 中，檢查 `weekly_forecasts` 表是否已包含 `pop` 欄位，若為舊版結構則安全刪除重建。
- **一週天氣預報表結構擴充**：在 `weekly_forecasts` 中新增 `pop` (降雨機率, INTEGER)、`uv_index` (紫外線指數, TEXT)、`ci` (體感舒適度, TEXT) 與 `wind` (風速風向, TEXT) 欄位。
- **CWA API 解析器升級**：在 `sync_weekly_data_internal()` 中：
  - 擷取 `12小時降雨機率` 並轉換為 `pop`。
  - 擷取 `最大舒適度指數Description` 並轉換為 `ci`。
  - 擷取每隔一日 daytime 時段的 `紫外線指數` 並映射為 `uv_index`（夜間時段為 `-`）。
  - 合併 `風向` 與 `風速` (蒲福風級) 轉換為 `wind`。

#### [MODIFY] [ai_helper.py](file:///f:/HW10-20260702-天氣預報/ai_helper.py)
- **SSL 驗證控制整合**：
  - 引入環境變數 `DISABLE_SSL_VERIFY` 控制開關。
  - 針對自訂 API 呼叫的 `requests.post()`，傳入 `verify=not _DISABLE_SSL` 參數。
  - 針對 OpenAI SDK，如果啟用了停用 SSL 驗證，則以 `httpx.Client(verify=False)` 初始化 OpenAI 客戶端以安全繞過 SSL 證書錯誤。
- **Prompt 富文本升級**：在 `generate_weekly_weather_advice()` 中，將一週預報的降雨機率、紫外線、體感舒适度與風力等新欄位加入 Prompt 資料結構，讓 AI 寫出的週報內容更加具體且專業。
- **Fallback 本地規則引擎優化**：在 `generate_mock_weekly_advice()` 中，引入對 UV 指數與強風的邏輯判斷，在無 Key 模式下提供更貼近真實氣象的防曬與防風提示。

#### [MODIFY] [app.py](file:///f:/HW10-20260702-天氣預報/app.py)
- 配合 `weather.py` 重建與欄位擴充，確保 API 路由 `/api/weekly_weather` 能正確輸出新欄位 JSON 供前端使用。

#### [MODIFY] [templates/index.html](file:///f:/HW10-20260702-天氣預報/templates/index.html)
- **整合 Chart.js 溫度趨勢折線圖**：
  - 引進 Chart.js 庫 (CDN)。
  - 在頁籤切換與預報卡片上方，新增「溫度變化趨勢圖」毛玻璃區塊。
  - 渲染一條雙線折線圖（最高溫紅橘色漸層，最低溫藍青色漸層），格線採用精緻微透明白線，與毛玻璃暗色氛圍完美契合。
  - 當切換縣市或切換 36h/一週頁籤時，圖表會流暢地進行動畫切換與數據重繪。
- **Windy 地圖觀測圖層控制**：
  - 在 Windy 地圖標題旁，新增精緻的切換按鈕組：「即時風場 (Wind)」、「天氣雷達 (Radar)」、「氣溫分佈 (Temp)」、「雲量觀測 (Clouds)」。
  - 點擊按鈕時，動態更新 Windy Iframe 的 `overlay` 參數，讓地圖展示相應的觀測圖層。
- **未來一週預報卡片展示升級**：
  - 將卡片內的降雨狀態視覺替代（有雨/無雨）升級為「真實降雨機率 (pop%)」顯示。
  - 新增「體感舒適度 (ci)」與「紫外線指數 (UV)」精美小圖示與文字，讓資訊層次更豐富。
